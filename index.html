한 줄 결론 — 셋은 경쟁이 아니라 겹치는 교집합을 가진 한 식구예요. 웹앱·배포는 코드, 문서·행정·연동은 코워크, 가벼운 질문은 챗 — 상황 따라 골라 쓰면 됩니다. 굳이 하나만 고집할 필요 없어요!

클로드 코드 데스크탑 설치하기

버튼 몇 번이면 끝납니다. 검은 명령창은 아직 안 만나도 돼요.

① 다운로드 페이지 열기

claude.ai/download 으로 들어갑니다. (검색창에 "Claude 다운로드"로 찾아도 돼요)

② Windows용 받기

Download for Windows 버튼을 누르면 설치 파일(.exe)이 다운로드 폴더에 받아집니다.

③ 설치 파일 실행

받은 파일을 더블클릭 → 안내대로 다음 · 설치. "알 수 없는 게시자" 경고가 나오면 추가 정보 → 실행을 누르세요.

④ 로그인

설치가 끝나면 자동으로 열립니다. 챗에서 쓰던 계정 그대로 로그인하면 끝! (구글 로그인 가능)

⑤ 작업 폴더 고르기

"어느 폴더에서 일할까요?"라고 물으면 바탕화면에 새 폴더를 하나 만들어 골라주세요. 이 폴더가 클로드의 작업실이 됩니다.

✅

설치 끝! 이 단계까지만 해도 간단한 자리 바꾸기 추첨 웹페이지 하나 만들어줘 처럼 부탁할 수 있어요. 4·5·6번의 배포 도구는 인터넷에 올릴 때만 필요하니, 그때 깔아도 됩니다.

기본 사용 설명서

챗이랑 거의 똑같아요. 그냥 한국어로 말하면 됩니다.

💬기본은 "그냥 말 걸기"

화면 아래 입력창에 한국어로 부탁하면 됩니다. 챗과 다른 점은, 답만 하는 게 아니라 내 폴더에 진짜 파일을 만든다는 것뿐이에요. 아래 문장은 클릭하면 복사돼요 👇

  • 우리 반 자리 바꾸기 추첨 웹페이지 하나 만들어줘
  • 방금 만든 파일에서 글씨 더 크게 하고 색깔 파란색으로 바꿔줘
  • 이 폴더에 뭐가 있는지 정리해서 알려줘

⏎알아두면 편한 기본 동작

  • 승인 요청 — 파일을 바꾸거나 명령을 실행하기 전에 "해도 될까요?"라고 물어요. Yes를 누르면 진행됩니다. (자주 쓰는 건 "항상 허용")
  • 중간에 멈추기 — 방향이 틀어졌다 싶으면 Esc 키로 즉시 멈추고 다시 지시할 수 있어요.
  • 결과 미리보기 — 만든 HTML 파일은 다운로드 폴더에서 더블클릭하면 바로 브라우저로 열려요.

⚡슬래시(/) 명령 — 단축 기능

입력창에 / 를 치면 편리한 기능 목록이 뜹니다. 외울 필요 없이 슬래시만 쳐서 골라 쓰면 돼요.

명령이럴 때
/clear대화가 길어져 헷갈릴 때 — 깨끗하게 새로 시작
/code-review만든 코드에 문제 없는지 점검받고 싶을 때
/mcp외부 도구(Netlify·GitHub 등) 연결·로그인 관리 — 6번에서 사용
💬

명령어, 직접 안 쳐도 돼요 — 아래 4·5·6번에 까만 명령어 박스가 나오는데, 외우거나 직접 칠 필요 없어요. 그냥 클로드에게 "필요한 거 알아서 깔고, 막히면 단계별로 알려줘" 라고 말하면 클로드가 뭐가 없는지 보고 차례로 설치·실행합니다. 까만 박스는 "잘 안 될 때 참고용"이에요.
단, 계정 가입 · 브라우저 로그인 · 구글 "확인 안 됨" 동의 화면처럼 직접 눌러야 하는 것만 선생님이 해주면 됩니다.

⚠️

한 가지만 기억하세요 — 클로드는 현재 작업 폴더 안에서만 일합니다. 중요한 파일은 백업해두고, 처음엔 비어 있는 연습용 폴더에서 시작하는 걸 추천해요.

4

GAS(구글 앱스스크립트) 앱 바로 배포하기

구글 시트와 연결되는 학교 앱(점수판·신청서 등)을 클로드가 직접 올립니다.

GAS는 구글 계정만 있으면 무료 서버처럼 쓸 수 있는 도구예요. 보통은 브라우저에서 코드를 복붙해야 하지만, clasp라는 도구를 깔면 클로드가 내 PC에서 곧장 구글로 업로드·배포합니다.

📦딱 한 번만: 도구 2개 설치

① Node.js 설치 (clasp가 돌아가는 엔진)

nodejs.org에서 초록 LTS 버튼을 받아 설치하거나, 클로드에게 아래 명령을 시키세요(클릭 복사):

Node.js 설치 (Windows)
winget install -e --id OpenJS.NodeJS.LTS
② clasp 전역 설치
npm install -g @google/clasp

🔑구글 계정 연결 (한 번만)

🔓

로그인 전에 한 번만 — script.google.com/home/usersettings 접속 → Apps Script API 스위치를 켜짐(On)으로 바꿔주세요.

로그인 — 브라우저가 열리면 구글 계정 선택
clasp login

🚀이제부터는 클로드가 다 합니다

앱을 만든 뒤 이 앱 구글 앱스스크립트로 배포해줘 한마디면, 클로드가 아래 명령들을 순서대로 실행합니다. (참고용)

클로드가 자동으로 실행하는 배포 흐름
# 1) 새 스크립트 프로젝트 만들기
clasp create-script --type standalone --title "우리반 점수판"

# 2) 내 PC의 코드를 구글로 업로드
clasp push --force

# 3) 웹앱으로 배포 → 접속 주소(URL)가 나옵니다
clasp create-deployment --description "v1 공개"
🤖

이 부분은 신경 안 써도 돼요 — 웹앱으로 "진짜 작동"하려면 설정 파일(appsscript.json)에 웹앱 항목을 넣고 "누구나 접근 가능"으로 배포하는 등 자잘한 작업이 있는데, 이건 전부 클로드가 알아서 처리합니다. 교사는 이 앱 웹앱으로 배포하고 누구나 접속되게 해줘 라고만 하면 돼요.

🛡️

딱 한 번 나오는 구글 경고 — 처음 배포·실행할 때 구글이 "이 앱은 확인되지 않았습니다(Google hasn't verified this app)" 화면을 띄울 수 있어요. 내가 만든 내 앱이니 안전합니다. 고급(Advanced) → OOO(으)로 이동(Go to … unsafe) → 허용(Allow)을 차례로 누르면 끝. 이건 누구 컴퓨터에서든 똑같이 한 번만 거치면 됩니다.

🔗

완성! 배포가 끝나면 https://script.google.com/macros/s/.../exec 형태의 주소가 나옵니다. 이 링크를 공유하면 끝. 수정 후엔 다시 배포해줘 만 하면 됩니다.

GitHub에 올리기 (+ 무료 웹사이트 만들기)

코드를 안전하게 보관하고, GitHub Pages로 공짜 홈페이지까지. 방법 2가지.

GitHub는 코드를 보관·공유하는 창고예요. GitHub Pages 기능으로 만든 HTML을 무료 웹사이트로도 띄울 수 있습니다. 먼저 github.com/signup 에서 무료 가입부터 하세요.

A 명령어로 올리기 (gh CLI)토큰 없이 제일 쉬움 초보자 추천

① gh(깃허브 도구) 설치 — 클로드에게 시키기:

GitHub CLI 설치 (Windows)
winget install -e --id GitHub.cli

② 로그인 — 실행하면 질문이 나와요: GitHub.com → HTTPS → Login with a web browser를 고르고, 화면에 뜬 8자리 코드를 복사해 붙여넣으면 끝.

로그인
gh auth login

③ 업로드 — 이제 이 폴더 깃허브에 새 저장소로 올려줘 한마디면 클로드가 아래를 실행합니다.

클로드가 자동 실행하는 업로드 흐름
# 이 폴더를 git 저장소로 초기화 후 깃허브에 한 번에 업로드
git init
git add .
git commit -m "첫 업로드"
# 브랜치 이름을 main 으로 통일 (Pages 설정과 맞추기)
git branch -M main
gh repo create my-school-app --public --source=. --push
B GitHub MCP로 연결하기토큰(PAT)이 필요해 조금 더 복잡
🔎

"GitHub MCP가 안 보이던데?" — 맞아요. GitHub MCP는 커넥터 목록(갤러리)에 없어서 검색으로는 안 나와요. 아래처럼 개인 토큰(PAT)과 함께 명령어로 직접 추가하는 게 가장 확실합니다. (브라우저 로그인 방식도 있지만 프로그램마다 달라서, 클로드 코드에선 토큰 방식이 제일 잘 됩니다.) 그냥 코드 올리기만 할 거면 방법 A(gh CLI)가 훨씬 간단하니 그걸 추천해요.

① 토큰(PAT) 발급 — github.com/settings/tokens → Generate new token (classic) → 이름 입력 · 만료기간 선택 · repo 항목 체크 → Generate → 나온 토큰 복사 (이 화면에서만 보여요!)

② 연결 — 아래에서 여기에_내토큰 자리에 ①의 토큰을 붙여넣어 실행:

GitHub MCP 연결 (토큰 필요)
claude mcp add-json github '{"type":"http","url":"https://api.githubcopilot.com/mcp","headers":{"Authorization":"Bearer 여기에_내토큰"}}'
✅

연결 확인은 /mcp 또는 claude mcp list. 잘 붙으면 그다음부터는 깃허브에 my-school-app 저장소 새로 만들고 이 파일들 올려줘 처럼 말로 시키면 됩니다.

공식 문서: github.com/github/github-mcp-server

🌐무료 웹사이트로 띄우기 (GitHub Pages)

올린 HTML을 실제 주소로 공개하려면, 가장 쉬운 건 웹에서 켜기예요:

  1. 내 저장소 페이지 → 위쪽 Settings 탭
  2. 왼쪽 메뉴 Pages 클릭
  3. Source를 Deploy from a branch → 브랜치 main / 폴더 / (root) 선택 후 Save

또는 클로드에게 이 저장소 GitHub Pages 켜줘 라고 하면 아래 API 명령으로 대신 켜줍니다.

클로드가 Pages를 켜는 명령 (OWNER/REPO는 내 것으로)
gh api -X POST repos/OWNER/REPO/pages -f "source[branch]=main" -f "source[path]=/"
🎉

몇 분 뒤 https://(내아이디).github.io/my-school-app 주소로 누구나 접속할 수 있는 사이트가 완성됩니다.

6

Netlify로 배포하기

"진짜 웹주소"를 가장 빠르게 만드는 방법. 상황에 맞게 3가지 중 골라요.

Netlify는 HTML을 인터넷에 올려 주소를 만들어주는 무료 서비스예요. netlify.com 가입 후, 아래 셋 중 하나로 올리면 됩니다.

🧩

잠깐, MCP가 뭐예요? — MCP는 클로드에게 끼우는 "능력 확장팩"이에요. Netlify MCP를 한 번 연결하면, 클로드가 터미널 명령 없이 직접 배포·주소 확인·설정까지 합니다. 반복 배포할 거면 이게 제일 편해요(방법 B).

A 마우스로 드래그 (설치 0)딱 한 번만 올릴 때, 가장 쉬움

① Drop 페이지 열기

app.netlify.com/drop 접속.

② 파일 끌어다 놓기

만든 HTML 파일(또는 폴더)을 페이지 위로 마우스로 드래그.

③ 주소 받기

몇 초 뒤 접속 주소가 뜹니다. 끝!

🖱️

명령어도 설치도 없이 드래그 한 번. 일회성 공유엔 이게 최고예요.

B Netlify MCP로 연결하기클로드가 직접 배포·관리 반복 배포 추천

① 준비물 — Node.js(4번에서 설치)와 Netlify 계정. Netlify는 공식 MCP 서버(@netlify/mcp)를 제공합니다.

② 연결 — 클로드에게 아래 명령을 시키거나 /mcp 메뉴에서 추가:

Netlify MCP 연결
claude mcp add netlify -- npx -y @netlify/mcp
🔐

연결 후 /mcp → netlify → Authenticate로 브라우저 로그인 한 번. 그다음부터는 이 폴더 Netlify에 새 사이트로 배포하고 주소 알려줘 처럼 말로 시키면 클로드가 직접 올리고 주소를 알려줍니다.

공식 문서: docs.netlify.com · github.com/netlify/netlify-mcp

C 명령어로 배포 (Netlify CLI)MCP 대신 직접 명령으로
① CLI 설치 (한 번만)
npm install -g netlify-cli
② 로그인 (브라우저 인증)
netlify login
③ 현재 폴더를 정식(공개) 주소로 배포
netlify deploy --prod
💬

처음 한 번은 질문이 나와요 — Create & configure a new site를 고르고, 배포 폴더를 묻거든 마침표(.)를 입력. 클로드가 옆에서 골라줍니다.

7

클로드 코드 데스크탑 꿀팁 모음

알면 훨씬 편해지는, 비전공자용 실전 팁.

🧹

헷갈리면 /clear

대화가 길어지면 클로드도 헷갈려요. 새 작업 시작 땐 /clear로 비우고 시작.

🛑

이상하면 Esc

엉뚱한 방향이면 Esc로 즉시 멈추고 "아니, 이렇게 말고…"라고 다시 지시.

🗂️

작업마다 새 폴더

앱 하나당 폴더 하나. 파일이 안 섞이고, 통째로 배포하기도 좋아요.

🧩

MCP로 능력 확장

Netlify·GitHub처럼 MCP를 연결하면 클로드가 그 서비스를 직접 다뤄요. /mcp에서 관리.

📸

화면을 붙여넣기

참고 디자인·오류 화면을 캡처해 입력창에 붙여넣으면 보고 작업해줘요.

🔍

완성 후 /code-review

중요한 앱이면 /code-review로 버그·보안을 한 번 점검받고 배포.

👀

승인은 천천히

파일 삭제·덮어쓰기 승인이 뜨면 내용을 한 번 보고 누르기. 급할 땐 "항상 허용".

🗣️

완벽한 지시는 NO

처음부터 길게 안 써도 돼요. 일단 시키고 결과 보며 대화로 다듬는 게 빠릅니다.

🌱

마지막 한마디 — 처음엔 작은 것부터. "반 이름표 만들기"처럼 가벼운 걸로 한 번 끝까지 배포해보면, 그 다음부터는 큰 앱도 두렵지 않아요. 막히면 그냥 클로드에게 이거 어떻게 해? 단계별로 알려줘 라고 물어보세요. 그게 가장 빠른 길입니다.
